📋 Queries
Q1
Select Query
Reads data — never changes anything
📖 READ ONLY · SAFE
Create› Query Design› Add Table› Drag Fields› ▶ Run
  1. Go to Create tab → click Query Design
  2. In "Show Table" popup: double-click your table → Close
  3. Double-click each field you want OR drag to the grid below
  4. In Criteria row: type condition e.g. "Alice" or >18 or Like "A*"
  5. Click ▶ Run (red ! button) to see results
🧠 Memory TrickSELECT = "Show me stuff" — like a search. Just READS, never touches data.
💡 Exam TipAdd Sort row = Ascending/Descending. Uncheck Show box to hide a field used only for filtering.
Q2
Make-Table Query
Creates a brand new permanent table
🏗️ ACTION · CREATES NEW TABLE
Query Design› Design tab› Make Table› Name it› ▶ Run
  1. Build a Select Query first (set fields + criteria)
  2. In Query Design tab → click Make Table
  3. In the popup: type a new table name e.g. TopStudents → OK
  4. Click ▶ Run → confirm the warning popup → Done!
  5. Check the new table appears in the left panel
🧠 Memory Trick"Make Table" = COPY & SAVE results into a new box. Like taking a photo of filtered data.
⚠️ Watch OutRe-running overwrites the old table with the same name — no undo!
Q3
Append Query
Adds rows from one table INTO another
➕ ACTION · ADDS ROWS
Query Design› Append› Choose Target› Map Fields› ▶ Run
  1. Open Query Design → add SOURCE table
  2. Click Append in the Design tab
  3. Popup: choose the TARGET (destination) table → OK
  4. A new row "Append To" appears — match source fields to target fields
  5. Click ▶ Run → confirm → rows are added to target
🧠 Memory TrickAppend = "POUR" data from one table into another. Source → pours into → Target.
💡 Exam TipBoth tables must have matching field types. PK field must not duplicate — it'll error.
Q4
Delete Query
Permanently removes matching records
🗑️ ACTION · PERMANENT DELETION
Query Design› Delete› Set Criteria› ▶ Run
  1. Open Query Design → add your table
  2. Click Delete in the Design tab
  3. Drag the * (asterisk = all fields) into the grid — this targets whole rows
  4. Also drag the field you'll filter by, set Criteria e.g. <"01/01/2020"
  5. Click ▶ Run → read warning carefully → confirm
⚠️ DANGER — No Undo!Always run as a SELECT first to preview what will be deleted. Only then switch to Delete.
🧠 Memory TrickDelete = DESTROY. Run Select first, verify rows, THEN delete. "See before you kill."
Q5
Update Query
Modifies values in existing records
✏️ ACTION · CHANGES EXISTING DATA
Query Design› Update› Update To row› ▶ Run
  1. Open Query Design → add table
  2. Click Update in the Design tab
  3. An "Update To" row appears below Field row
  4. Drag the field to update → in "Update To" type new value or formula:
    [Price]*1.1 ← increase price by 10%
  5. Optionally set Criteria to limit which rows update
  6. Click ▶ Run → confirm
🧠 Memory Trick"Update To" = the NEW VALUE. Think: "Update [field] To [new value] Where [criteria]."
💡 Exam TipYou can reference the same field: [Salary]+500 adds 500 to current salary.
Q6
Crosstab Query
Summarises data in a row × column matrix
📊 READ · PIVOT-LIKE GRID
Create› Query Wizard› Crosstab› Set Row/Col/Value
  1. Go to Create → Query Wizard → choose Crosstab Query Wizard
  2. Select the table/query to use → Next
  3. Choose Row Heading field (e.g. Month) → Next
  4. Choose Column Heading field (e.g. Product) → Next
  5. Choose Value field + function (e.g. Sum of Sales) → Next
  6. Name the query → Finish
🧠 Memory TrickROW = "what goes down left" · COL = "what goes across top" · VALUE = "numbers in the middle"
💡 Exam TipCan also do manually: Query Design → Crosstab button → set Crosstab row to "Row Heading / Column Heading / Value"
Q7
Parameter Query
Asks user for input before running
❓ DYNAMIC · USER-DRIVEN
Select Query› Criteria row› Type [prompt]› ▶ Run
  1. Build a Select Query as normal
  2. In the Criteria row of the field to filter, type your prompt inside square brackets:
[Enter Student Name:]
[Enter Min Age:]
  1. Click ▶ Run — a popup appears asking for input
  2. User types value → results filter to match
🧠 Memory Trick[ ] square brackets = ASK THE USER. Whatever is inside becomes the popup question. No brackets = just criteria. With brackets = parameter.
💡 Exam TipCan combine: Between [Start Date:] And [End Date:]
Q8
Find Duplicates Query
Finds records with repeated values
🔍 WIZARD · FINDS REPEATS
Create› Query Wizard› Find Duplicates› Finish
  1. Go to Create → Query Wizard
  2. Choose "Find Duplicates Query Wizard" → Next
  3. Select the table to search → Next
  4. Pick the field(s) that might have duplicates (e.g. Name, Email) → Next
  5. Choose any extra fields to show → Next
  6. Name the query → Finish — runs and shows repeated rows
🧠 Memory Trick"Duplicates = Twins." The wizard finds all the twins (same value appearing more than once).
Q9
Find Unmatched Query
Finds records in A with no related record in B
🔗 WIZARD · FINDS ORPHANS
Create› Query Wizard› Find Unmatched› Set both tables› Finish
  1. Go to Create → Query Wizard → choose "Find Unmatched Query Wizard"
  2. Select Table A (the one you want results FROM, e.g. Customers) → Next
  3. Select Table B (the related table, e.g. Orders) → Next
  4. Pick the matching field in both tables (usually the FK/PK, e.g. CustomerID) → Next
  5. Select extra fields to show → Next → Name → Finish
🧠 Memory Trick"Unmatched = Lonely records." Customers with NO orders. Students with NO enrolments. The wizard finds who's been LEFT OUT.

📝 Forms
F1
Creating Basic Forms
User-friendly data entry screen
📥 INPUT · USER INTERFACE
METHOD 1 — Auto (Fastest)
Select Table› Create› Form
METHOD 2 — Wizard (More Control)
  1. Create → Form Wizard
  2. Pick table/query as data source → add fields using >>
  3. Choose layout: Columnar (one record) / Tabular (like spreadsheet) / Datasheet
  4. Choose style → name it → Finish
METHOD 3 — Design View (Most Control)
Create› Form Design› Add Field List
🧠 Memory TrickForm = FACE of the database. Users see the form, not the raw table. Columnar = one-at-a-time. Tabular = all rows.
💡 Exam TipTo switch views: Right-click form tab → Design / Form / Layout view.
F2
Bound & Unbound Controls
Connected vs standalone elements
BoundUnbound
Linked to?A table fieldNothing (standalone)
Changes data?Yes — edits tableNo — display only
ExampleName textboxTitle label, logo
Control Source= FieldNameEmpty
Set bound: In Design view → click control → Properties → Control Source → choose field
Check: If Control Source is empty = Unbound. Has a field name = Bound.
🧠 Memory TrickBound = BONDED to the table (like a chain). Unbound = FREE floating (like a sticky note on top).
💡 Exam TipCalculated control = also unbound. Control Source = =[Price]*[Qty]. Starts with = sign.
F3
List Box / Combo Box
Dropdown or fixed list for picking values
List BoxCombo Box
Looks likeAlways open listDropdown (click to open)
User can type?No — pick onlyYes — type or pick
Space usedMore (always visible)Less (collapses)
Best forSmall fixed listsLong lists, most forms
How to Add (Design View):
  1. Open form in Design View
  2. In Design tab → Controls group → click List Box or Combo Box icon
  3. Draw it on the form — Wizard launches automatically
  4. Choose source: Table/Query or type values yourself
  5. Set Bound Column (which column value gets saved) → Finish
🧠 Memory TrickCombo = Combination of textbox + list (you can type OR pick). List = List only, always visible, pick only.
📄 Reports
R1
Creating Reports
Formatted, printable data output
📤 OUTPUT ONLY · READ-ONLY
METHOD 1 — Auto (Fastest)
Select Table/Query› Create› Report
METHOD 2 — Wizard
  1. Create → Report Wizard
  2. Choose table/query → add fields using >> → Next
  3. Add Grouping levels (optional) → Next
  4. Set Sort order (up to 4 fields) → Next
  5. Choose Layout: Tabular / Columnar / Justified → orientation → Next
  6. Name the report → Finish
🧠 Memory TrickReport = RECEIPT. You print it out and show it. Nobody types into it. It's the READ-ONLY output version of a form.
💡 Exam TipTo edit: right-click report → Design View. To preview: right-click → Print Preview.
R2
Sorting & Grouping
Organise records & add subtotals
SortGroup
What it doesOrders records A→Z or Z→AClusters records by a field value
ResultRecords in orderRecords under a heading
ExampleStudents A→Z by NameStudents grouped by Course
How to Set in Design View:
  1. Open report in Design View
  2. In Design tab → click Group & Sort
  3. A panel opens at the bottom — click Add a group or Add a sort
  4. Choose the field → set Ascending / Descending
  5. For grouping: expand options → set Group Header / Group Footer (for subtotals)
  6. Add a Text Box in Group Footer → Control Source = =Sum([Salary])
🧠 Memory TrickSort = like alphabetising a list. Group = like putting things in folders. Sort first (A→Z), then Group (by category).
💡 Exam TipFunctions in footer: =Sum()  =Count()  =Avg()  =Max()  =Min()
R3
Header & Footer
5 section zones of a report
📌 Report Header — appears ONCE at very top (title, logo)
📄 Page Header — repeated at top of EVERY page (column labels)
⬛ [Group Header] — optional, appears before each group
▤ Detail — the actual data records (repeats per row)
⬛ [Group Footer] — optional, subtotals per group
📄 Page Footer — repeated at bottom of EVERY page (=Page())
📌 Report Footer — appears ONCE at very end (grand total)
How to show/hide sections:
Design View› Arrange tab› Page Header/Footer
  1. In Design View: right-click any section → toggle Header/Footer
  2. Add Label to Report Header → type your title
  3. Add Text Box to Page Footer → ="Page " & [Page] & " of " & [Pages]
  4. Add Text Box to Report Footer → =Sum([Salary])
🧠 Memory TrickReport H/F = Once each (sandwich bread at start & end). Page H/F = Every page (like a TV show intro that repeats each episode).
