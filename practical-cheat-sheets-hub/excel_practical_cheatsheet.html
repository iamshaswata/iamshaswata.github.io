🔵 Core Functions
F1
IF Function
Returns one of two values based on a test
🔀 LOGICAL TEST
=IF(logical_test, value_if_true, value_if_false)
ArgumentMeaningExample
logical_testCondition to checkA2>50
value_if_trueReturn if TRUE"Pass"
value_if_falseReturn if FALSE"Fail"
=IF(A2>50, "Pass", "Fail")
← if A2=75 → "Pass"

=IF(B2>=18, "Adult", "Minor")
← nested: =IF(A2>=80,"A",IF(A2>=60,"B","C"))
🧠 Memory TrickRead it as a question: "IF A2 > 50, say 'Pass', else say 'Fail'". Three parts: TEST → TRUE → FALSE. Always in that order.
💡 Exam TipText values need quotes: "Pass". Numbers don't: 0. Nested IF = IF inside IF for multiple grades.
F2
COUNT Functions
Count cells — 4 variants to know
🔢 COUNTING
FunctionCountsExample
COUNTNumbers only=COUNT(A1:A10)
COUNTANon-empty cells (any type)=COUNTA(A1:A10)
COUNTBLANKEmpty cells only=COUNTBLANK(A1:A10)
COUNTIFCells matching criteria=COUNTIF(A1:A10,">"&50)
=COUNT(A1:A10)7 ← 7 number cells
=COUNTA(A1:A10)9 ← 9 non-empty
=COUNTBLANK(A1:A10)1 ← 1 empty cell
🧠 Memory TrickCOUNT = numbers only. COUNTA = A for "All" (text + numbers). COUNTBLANK = blanks (empty boxes).
F3
COUNTIF
Count cells that match a condition
🎯 CONDITIONAL COUNT
=COUNTIF(range, criteria)
Count "Pass" in B1:B20:
=COUNTIF(B1:B20, "Pass")

Count values greater than 50:
=COUNTIF(A1:A20, ">50")

Count matching a cell value:
=COUNTIF(A1:A20, D1)

Wildcard — starts with "A":
=COUNTIF(A1:A20, "A*")
🧠 Memory TrickCOUNTIF = "Count IF it matches." Two things only: WHERE to look (range) and WHAT to match (criteria). Criteria with >< must be in quotes.
💡 Exam Tip">50" — the operator goes INSIDE the quotes when hardcoded. Use ">"&D1 to reference a cell.
F4
SUM / AVERAGE
Add up or find the mean of a range
➕ ARITHMETIC
=SUM(A1:A10) ← adds all values
=SUM(A1, B1, 50) ← add specific cells + number

=AVERAGE(A1:A10) ← mean of range

Conditional versions:
=SUMIF(A1:A10,">50",B1:B10) ← sum B if A>50
=AVERAGEIF(A1:A10,"Pass",B1:B10)
🧠 Memory TrickSUM = total. AVERAGE = total ÷ count. SUMIF = "Add only the qualifying ones." Same structure as COUNTIF but adds instead of counts.
💡 Exam TipQuick shortcut: select range → look at Status Bar at bottom — Sum & Average shown automatically without formula.
F5
MIN / MAX
Find the smallest or largest value
⬆⬇ EXTREMES
=MIN(A1:A20) ← lowest value in range
=MAX(A1:A20) ← highest value in range

Works across multiple ranges:
=MIN(A1:A10, C1:C10)
=MAX(A1:A10, C1:C10)

SMALL & LARGE (nth value):
=SMALL(A1:A20, 2) ← 2nd smallest
=LARGE(A1:A20, 3) ← 3rd largest
🧠 Memory TrickMIN = Minimum = smallest (bottom of the mountain). MAX = Maximum = largest (top of the mountain). MIN ignores text.
💡 Exam TipSMALL(range, 1) = same as MIN. LARGE(range, 1) = same as MAX. Use SMALL/LARGE when you need 2nd, 3rd rankings.
F6
ROUND
Round to a specified number of decimal places
🔄 ROUNDING
=ROUND(number, num_digits)
num_digitsResult for 3.14159
23.14 ← 2 decimal places
03 ← nearest whole number
-10 ← nearest 10
-2100 ← nearest 100 (for 149.5)
=ROUND(A2, 2) ← round to 2 dp
=ROUNDUP(A2, 0) ← always rounds UP
=ROUNDDOWN(A2, 0) ← always rounds DOWN
🧠 Memory TrickPositive digits = decimal places (right of dot). Zero = whole number. Negative = rounds LEFT of decimal (tens, hundreds). Think: positive = zoom in, negative = zoom out.

VLOOKUP
Vertical lookup — searches down a column
⬇ VERTICAL SEARCH
=VLOOKUP(lookup_value, table_array, col_index, range_lookup)
ArgumentWhat to putExample
lookup_valueWhat to search forA2 or "Alice"
table_arrayThe table range (lock with $)$A$1:$D$20
col_indexWhich column to return (1=first)3
range_lookupFALSE=exact, TRUE=approximateFALSE
Find salary of student in A2:
=VLOOKUP(A2, $B$1:$D$20, 3, FALSE)
↑ search A2 in col B, return value from col D (3rd col of range)

🧠 Memory TrickV = Vertical = searches DOWN a column. Think: "Find it, then step right N columns." Always use FALSE for exact match in exams. $ lock the table!
⚠️ Common MistakeLookup value must be in the FIRST column of your table_array. Column index 1 = the first column of the range (not column A of the sheet).
💡 Exam TipIf result shows #N/A = value not found. If #REF! = col_index too high. If #VALUE! = wrong data type.

💰 Financial Functions
FIN1
FV — Future Value
How much an investment is worth in the future
📈 SAVINGS / INVESTMENT GROWTH
=FV(rate, nper, pmt, [pv], [type])
ArgumentMeaningExample
rateInterest rate per period5%/12 (monthly)
nperTotal number of periods5*12 (5 yrs monthly)
pmtPayment each period (negative = paying out)-500
pvPresent value (lump sum today, optional)-10000
type0=end of period (default), 1=start0
Save £500/month at 5%/yr for 5 years:
=FV(5%/12, 60, -500)£34,040
🧠 Memory TrickFV = "Future Value" = what your piggy bank will hold LATER. Payments are negative (money going OUT of your pocket). Result is positive (money coming IN future).
💡 Rate RuleAnnual rate ÷ 12 = monthly. Annual rate ÷ 4 = quarterly. Always match rate period to nper period!
FIN2
PV — Present Value
What a future amount is worth TODAY
📉 VALUE TODAY
=PV(rate, nper, pmt, [fv], [type])
How much to invest today to get £10,000 in 5 yrs at 6%:
=PV(6%/12, 60, 0, 10000)-£7,414
← negative = amount you pay TODAY
FVPV
Question"What will I have?""What's it worth now?"
KnownPayments + rate todayFuture amount + rate
FindsFuture lump sumToday's equivalent value
Think of it asSaving forwardDiscounting backward
🧠 Memory TrickPV = "Price today". FV = future. PV = present. They're inverses. FV asks "how much WILL I have?" PV asks "how much is that WORTH today?"
FIN3
PMT — Payment / EMI
Monthly loan payment calculation
💳 LOAN EMI CALCULATOR
=PMT(rate, nper, pv, [fv], [type])
ArgumentMeaning
rateInterest rate per period (annual ÷ 12 for monthly)
nperTotal number of payments (years × 12 for monthly)
pvLoan amount (present value — positive)
£100,000 loan, 8% annual, 20-year term:
=PMT(8%/12, 20*12, 100000)-£836/month
← negative = you pay OUT each month

To show as positive, wrap in ABS():
=ABS(PMT(8%/12, 240, 100000))£836
🧠 Memory TrickPMT = "Pay Monthly Total" = your EMI. Three MUST-KNOW things: rate/12 (monthly interest), years×12 (total months), loan amount. Always negative (you're paying out).
💡 Exam TipRelated: =IPMT(rate,per,nper,pv) = interest part of payment. =PPMT(rate,per,nper,pv) = principal part.
FIN4
Depreciation: SLN · DB · SYD
How an asset loses value over time
📦 ASSET VALUE DECLINE
FunctionMethodSyntax
SLN Straight-Line
Equal amount every year		 =SLN(cost, salvage, life)
DB Declining Balance
Higher depreciation early, less later		 =DB(cost, salvage, life, period)
SYD Sum-of-Years Digits
Accelerated — more depreciation early		 =SYD(cost, salvage, life, per)
Asset: cost=£10000, salvage=£1000, life=5 years:
=SLN(10000, 1000, 5)£1,800/yr ← same every year

=DB(10000, 1000, 5, 1)£3,690 yr1 ← year 1 (period=1)

=SYD(10000, 1000, 5, 1)£3,000 yr1 ← year 1 of 5
ArgumentMeaning
costOriginal purchase price
salvageValue at end of useful life (scrap value)
lifeNumber of periods (years)
period/perWhich year to calculate (1, 2, 3...)
🧠 Memory TrickSLN = "Straight Line = Same amount" (like equal slices of pie). DB = "Declining Balance = Drops fast" (like a new car). SYD = "Sum Years Digits = Slides down" (accelerated, middle ground).

Summarise large datasets instantly
📋 DYNAMIC SUMMARY TABLE
Select Data› Insert› PivotTable› OK› Drag Fields
  1. Click anywhere inside your data table
  2. Go to Insert → PivotTable
  3. Choose "New Worksheet" or "Existing" → OK
  4. In the PivotTable Fields panel (right side), drag fields to 4 zones:
🔽 Filters
Filter the whole table
e.g. "Year = 2024"
→ Columns
Goes across top
e.g. Quarter, Month
↓ Rows
Goes down left side
e.g. Product, Region
Σ Values
Numbers in the middle
e.g. Sum of Sales
  1. To change aggregation: click value field → Value Field Settings → choose Sum / Count / Average
  2. To refresh after data changes: right-click pivot → Refresh
🧠 Memory TrickRows = what goes down. Columns = what goes across. Values = the numbers inside. Think of it as a T-shirt: Row = spine (down), Col = shoulders (across), Values = the logo in the middle.

T3
Chart Types
Pie · Column · Bar — when to use each
ChartBest ForExample Use
🥧 Pie Parts of a whole (%) — max 6 slices Market share, budget breakdown
📊 Column Comparing values over time (vertical bars) Monthly sales, annual growth
📉 Bar Comparing categories (horizontal bars) Product comparison, rankings
📈 Line Trends over time (continuous data) Stock prices, temperature
🔵 Scatter Relationship between 2 variables Height vs weight, XY data
How to Insert a Chart:
Select Data› Insert› Charts group› Pick type
💡 Exam TipChange chart type anytime: right-click chart area → Change Chart Type. Move chart to new sheet: right-click → Move Chart → New Sheet.
🧠 Memory TrickPie = proportions (one whole). Column = compare over time (stands up like a timeline). Bar = compare things (lying down like a ranking list).

Date & Time Formatting
Display and calculate with dates
Format CodeResult
dd/mm/yyyy25/12/2024
d-mmm-yy25-Dec-24
dddd, d mmmm yyyyWednesday, 25 December 2024
hh:mm:ss14:30:05
hh:mm AM/PM02:30 PM
dd/mm/yyyy hh:mm25/12/2024 14:30
Useful date functions:
=TODAY() ← today's date
=NOW() ← today's date AND time
=DAY(A1) ← extracts day number
=MONTH(A1) ← extracts month number
=YEAR(A1) ← extracts year
=DATEDIF(A1,B1,"Y") ← years between dates
🧠 Memory TrickDates are just NUMBERS in disguise (days since 1 Jan 1900). d=day, m=month, y=year, h=hour. Double = full (dd=01, d=1). Triple m = text (mmm=Jan, mmmm=January).
💡 Exam TipDays between two dates: just subtract! =B1-A1 then format as Number (not date). =NETWORKDAYS(A1,B1) = working days only.
